Top 3 Critical Findings

  1. Massive missing data (45-55% per column): Five columns (Vendor_Name, Unit_Price, Total_Amount, Ship_Date, Status) have 31-55% missing values. These were whitespace-only strings in the raw data, now exposed as true NaN.
  2. Extreme outliers: Total_Amount contains values up to $20M against a typical range of $0-$27K. Unit_Price has a $2M entry. These are likely data entry errors.
  3. 4 negative financial values: Total_Amount contains 4 negative entries (-$0.05 to -$1,149) in a column that should be positive-only.
Impact Statement: With 70% completeness and extreme outliers, this dataset cannot be used reliably for aggregate financial analysis (revenue totals, averages) without further source-system recovery. Row-level analysis on the 62 complete records is feasible. The 43 duplicate rows that were removed were inflating all counts by ~23%.

Top 3 Recommended Actions

  1. Recover missing data from source ERP (Effort: High) — 75 rows lack Unit_Price and Total_Amount. These must come from the source system.
  2. Investigate $2M+ outliers (Effort: Low) — 6 extreme values should be verified against actual invoices. Likely missing decimal points.
  3. Resolve 17 duplicate Order_IDs (Effort: Medium) — Same Order_ID with different data suggests merge/versioning issues at the source.
